## 냥습
```
냥습/A (A는 문자열)
```
학습데이터 A가 있는지 확인합니다.

<br>

```
냥습/A/B (1 < |A|, 0 < |B|, A, B는 문자열)
```
메시지 A를 보내면 브라운냥이 B를 말하도록 학습시킵니다. DB에 학습데이터 A가 저장됩니다.
- A, B는 다음을 만족해야 합니다.
  - A가 냥습금지어가 아님
  - A 안에 냥습금지기호가 없음
  - B 안에 냥습금지기호가 없음
  - A의 앞뒤가 공백이 아님
- B 안에 아래 문법들을 적용할 수 있습니다.
  - `[m]`: 자신을 지칭합니다.
  - `[y]`: 상대방을 지칭합니다.
  - `% A1, A2, ..., An %`: A1 ~ An 중 하나를 선택합니다.

<br>

```
냥습/목록
냥습목록
```
학습 목록을 확인합니다.

<br>

```
삭제/A (A는 문자열)
```
학습데이터 A를 삭제합니다.

<br>

## 명제
```
A -> B
A → B
A은[는] B(이)다!
A은[는] B이[가] 아니다! (A, B는 문자열)

(정규식으로 표현했을 경우)
^!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+ *-> *!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+$
^!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+ *→ *!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+$
^!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+[은는] +!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+이?다!$
^!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+[은는] +!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+[이가] +아니다!$
```
명제의 참\/거짓을 판별합니다.
- A, B는 명사로 해석합니다.
- A, B가 모음으로 끝나는지의 여부에 따라 올바른 조사를 붙여야 합니다.
  ```
  (모음)+는
  (자음)+은

  (모음)+(이)다!
  (자음)+이다!

  (모음)+가 아니다!
  (자음)+이 아니다!
  ```
- 아래 기호들은 모음으로 끝나는 글자로 해석합니다.
  - `_`
  - `2`, `4`, `5`, `9`
  - `L`, `M`, `N`, `R`을 제외한 알파벳 대문자
  - `l`, `m`, `n`, `r`을 제외한 알파벳 소문자
  - `ㅏ` ~ `ㅣ`
- 명제의 대우, 삼단논법이 자동으로 적용됩니다.
- 위의 정규식에 나와 있는 것처럼 A 또는 B 앞에 `!`(부정 연산자)를 붙일 수 있습니다.

<br>

```
명제/A/B (A, B는 문자열)
```
참 명제 `A -> B`가 있는지 확인합니다.

<br>

```
명제/목록
명제목록
```
참 명제 목록을 확인합니다.

<br>

```
명제/삭제/A/B (A, B는 문자열)
```
참 명제 `A -> B`를 삭제합니다.

<br>

```
명제/추가/A/B (0 < |A|, 0 < |B|, A, B는 문자열)
```
참 명제 `A -> B`를 추가합니다.
- A, B는 다음을 만족해야 합니다.
  - A, B가 정규식 `^!?[\wㄱ-ㅎㅏ-ㅣ가-힣]+$`을 만족함
  - A, B가 냥습금지어가 아님
